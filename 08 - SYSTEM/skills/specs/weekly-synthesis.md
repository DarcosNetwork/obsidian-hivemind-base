# weekly-synthesis

## Purpose

Synthesize the week's activity into one cohesive narrative: what moved, what stalled, what surprised, what to focus on next week. Unlike project-health (per-project, structured), this is cross-cutting — themes, patterns, and an honest "what actually happened" account.

## Trigger

**Manual:** "Synthesize this week" or "Generate weekly review" — invokes on demand.

**Scheduled:** Disabled by default. When enabled, fires Sundays at 19:00 Europe/Budapest (after `connection-finder` at 17:00, so the synthesis can reference new connections).

**Activation gate (CRITICAL):** this skill must NOT run on a sparse vault.
- `find "02 - NOTES/daily/" -name "*.md" -mtime -7 2>/dev/null | wc -l >= 5` (at least 5 daily notes from this week — gives the synthesis material to work with)

If fewer than 5 daily notes this week, skill aborts with: "weekly-synthesis requires ≥5 daily notes from this week. Currently N."

## Process

1. **Read the week's daily notes** (last 7 days, all in `02 - NOTES/daily/`).
2. **Read the week's Hermes outputs** at `06 - HERMES-OUTPUTS/{briefings,analyses,syntheses,reviews}/` filtered to last 7 days.
3. **Read the most recent `project-health` report** (if exists, for project-level context).
4. **Read the most recent `connection-finder` report** (if exists, for surfaced links).
5. **Read the previous weekly synthesis** at `06 - HERMES-OUTPUTS/syntheses/[prior-week]-weekly-synthesis.md` (if exists) — for continuity ("last week I said I'd focus on X; did I?").
6. **Extract:**
   - **Themes**: what topics dominated? (3-5 themes max, with concrete examples)
   - **Wins**: what actually shipped/completed this week? (1-5 wins, each with evidence)
   - **Failures**: what didn't happen that should have? (1-5 honest failures, no sugar-coating)
   - **Open loops**: tasks started but not completed, decisions deferred, projects paused
   - **Surprises**: anything Adam learned or noticed that wasn't on the radar
7. **Suggest focus for next week:** one specific outcome, not a list. Avoid "do more of X" framings — pick *one* thing.
8. **Quality Gate** before saving — see below.

```markdown
2. **[Theme name]** — ...
3. ...

## Wins

- ...

## Failures (honest)
- [What didn't happen] — [why, and what this signals]
- ...

## Open loops
- [Task/decision/project that's mid-flight, with next concrete step]
- ...

## Surprises
- [Anything Adam noticed this week that wasn't on the radar]

## Carry-over from last week
- [Items from previous synthesis that Adam said he'd address — did he? Yes/no, evidence.]

## Focus for next week
**One specific outcome.** Not a list. Pick the most leverage-y thing.

## Confidence note
- This synthesis is generated by `weekly-synthesis v0.1` based on daily notes and Hermes outputs from the week. It does not have access to conversations, meetings outside the vault, or anything Adam didn't capture. The "honest" framing above is grounded in what's in the vault — gaps in vault capture will show up as gaps here.
```

## Quality Gate

| Criterion | Check |
|---|---|
| **One sentence summary exists** | "The week in one sentence" section is exactly 1 sentence (1 period, no semicolons splitting it) |
| **Themes grounded in evidence** | Each theme cites at least one daily note or project note by name (e.g., "see [[2026-06-22-daily-...]]") |
| **Wins are concrete** | Each win has a deliverable or artifact, not "made progress" |
| **Failures are honest** | At least 1 failure if any open loops or stalled projects exist; no "all went great" framing when the data shows otherwise |
| **Focus is one item** | "Focus for next week" section has exactly 1 outcome, not 3 |
| **No hallucinated links** | Every `[[wikilink]]` in the synthesis resolves to an actual file in the vault |
| **Length in range** | 400-1200 words. Below 400 is too thin; above 1200 doesn't fit a Sunday-evening read |
| **Date headers correct** | `week-of` is the Monday of the week; `date` is the Sunday |

## Output

A single markdown file at `06 - HERMES-OUTPUTS/syntheses/YYYY-MM-DD-weekly-synthesis.md` (using the Sunday date). No other files modified.
